Création d'URL

Modèles d'URL

Brightcove Once UX utilise une syntaxe d'URL propriétaire basée sur la technologie Once VOD, permettant à la création d'URL d'être très similaire en fonction aux URL Once standard. Les seules différences dans la création d'URL pour Once UX sont un nom d'hôte mis à jour et l'inclusion de deux variables supplémentaires avant les variables typiques Once DeliveryType et RequestedFileType, décrites ci-dessous. Pour plus d'explications sur la création d'URL, consultez la section Mise en route avec une seule fois : Une fois la création d'URL.

Agrégation d'annonces VMAP

Les variables de contenu de distribution ci-dessus pour Once UX utilisent la méthode Ad Aggregation du service. Cette méthode récupère une liste de lecture publicitaire à partir de valeurs dérivées en résumant ou agréger les données de visualisation des annonces en fonction de votre applicationguid et du ou des serveurs publicitaires configurés. Lors de la réception de la réponse Once UX, cette méthode fournit des métadonnées et un objet VMAP contenant la charge publicitaire pour l'ensemble de la session de visualisation de la ressource qui a été normalisée dans VAST 3.0, puis incorporée à un objet XML VMAP 1.0. Cette réponse VMAP vous permet de gérer les fonctionnalités publicitaires côté client liées à l'expérience utilisateur (comptes à rebours de créations publicitaires ou de modules publicitaires, masquage des commandes du lecteur), de balisage et de types de créations supplémentaires (bannières, clics et publicités VPAID). Cet objet VMAP contient également un objet XML Brightcove Once, qui inclut les données Brightcove Once et une URL publicitaire insérée Once à utiliser pour la lecture. Voir le Exemple de réponse VMAP pour plus d'informations.

Paramètres d'annonce dynamique

Once UX permet l'utilisation de tous les paramètres d'annonces dynamiques (DAP) autorisés pour le remplissage des appels d'annonces et de balises de la même manière que la plate-forme Once. Pour obtenir des explications sur ces valeurs, reportez-vous à la section Getting Started with Once : Paramètres d'annonce dynamique.

Paramètres de chaîne de requête

Once UX utilise les mêmes paramètres de chaîne de requête que la plate-forme Once et trois sont couverts dans cette section qui peuvent être utilisés pour aider à affiner davantage la réponse et le comportement Once UX. Veuillez consulter la section Premiers pas avec une seule fois : Paramètres de chaîne de requête pour plus d'informations sur les autres paramètres de chaîne de requête disponibles.

Désactiver le suivi des événements

UMTP ={integer}

UMTP est utilisé pour remplacer ou désactiver le comportement par défaut d'impression vidéo et de suivi des événements côté serveur de Brightcove.

  • UMTP = 0

Par défaut, Brightcove déclenche des événements d'impression vidéo linéaire et de suivi sur le serveur publicitaire au nom du client / de l'application. Si le paramètre est défini explicitement, Once ne déclenchera pas les impressions ni les événements de suivi.

Sortie

sortie={integer}

Cette paire clé-valeur contrôle le format d'objet de la réponse renvoyée par Brightcove. Par défaut, Once UX répondra avec XML, donc output = 1 doit être utilisé si une réponse JSON est souhaitée.

  • XML = 0
  • JSON = 1

Timedtextextension

extension de texte temporel ={extension}

Utilisé pour modifier le type de légende de sortie pour timedtext Adaptive/M3u8 et Once UX <UO:TimedTexturl>Responses. Une fois que UX fournit un side-car par défaut dans un fichier xml au format DFXP dans la réponse VMAP pour les joueurs / applications / environnements qui peuvent ne pas être en mesure d'utiliser la méthode Apple de webvtt par défaut. Veuillez consulter l'exemple d'objet XML Once UX pour l'emplacement de ce fichier dans les réponses VMAP.

Les extensions disponibles sont les suivantes :

  • srt
  • vtt (WebVTT)
  • xml (DFXP utilisant une extension .xml)
  • dfxp

Explication des attributs VMAP:Adbreak

VMAP:Adbreak est un élément VMAP répétitif contenant des informations reçues par votre fournisseur d'annonces spécifiques à chaque publicité. Les AdBreaks seront toujours classés par ordre chronologique pour la position de la coupure publicitaire par heure dans le contenu (pre0, mid0, mid1, post0) et la séquence d'annonce dans une pause / pod (ad0, ad1, ad2).

Nœud Description
BreAKID La désignation VMAP de l'annonce. Cette valeur indique le type de pause, le titre du groupe de pause basé sur le décalage de temps et la valeur de position temporelle de base zéro de la pause.
décalage horaire Décalage temporel entre le début du contenu vidéo de base et le placement de la coupure publicitaire dans le contenu vidéo de base. Les pauses pré et Postroll afficheront respectivement les décalages de temps de début et de fin. Pour les coupures publicitaires midroll, le décalage horaire La valeur doit être combinée avec une durée cumulée de toutes les publicités affichées avant cette pause particulière pour un calcul précis de la pause. Pour les pauses publicitaires postroll, le début de la pause est longueur du contenu combinée à une durée cumulée de toutes les créations publicitaires précédemment affichées.
