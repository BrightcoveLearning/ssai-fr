Vue d'ensemble

Pour faciliter le débogage des publicités côté serveur, Dynamic Delivery with SSAI fournit des points de terminaison d'API pour suivre les appels d'annonces.

Notes :
  • Pour utiliser SSAI, votre compte Video Cloud doit être configuré pour la livraison dynamique et activé pour SSAI. Contactez votre gestionnaire de compte pour commencer à utiliser cette fonctionnalité.
  • Cette interface de débogage est réservée à la VOD SSAI.
  • Pour une liste des limitations, consultez le Aperçu de la SSAI document.

Pour déboguer des publicités côté serveur avec votre contenu vidéo stocké dans Video Cloud, procédez comme suit :

  1. Consultez les informations générales sur le chemin et l'autorisation de l'API
  2. Exécuter une trace pour une configuration d'annonce
  3. Récupérer les détails de la trace

Informations générales

Les informations suivantes concernent toutes les demandes d'API SSAI.

URL de base

L'URL de base de l'API SSAI est :

  https://ssai.api.brightcove.com/v1

Chemin du compte

Dans tous les cas, des demandes seront faites pour un compte Video Cloud spécifique. Ainsi, vous ajouterez toujours le terme accounts suivi de votre identifiant de compte à l'URL de base :

  https://ssai.api.brightcove.com/v1/accounts/your account id

Autorisation

Les requêtes API nécessitent un en-tête Authorization :

  Authorization: Bearer your access token

Le access_token est un jeton d'accès OAuth2 temporaire qui doit être obtenu à partir du service Brightcove OAuth. Pour plus d'informations sur la façon d'obtenir des informations d'identification client et de les utiliser pour récupérer des jetons d'accès, consultez la présentation OAuth de Brightcove.

Opérations

Lorsque vous demandez des informations d'identification client, vous devez spécifier le type d'accès au compte ou les opérations que vous souhaitez. Voici une liste des opérations actuellement prises en charge pour l'API SSAI :

  • Données SSAI :

    video-cloud/ssai/read
    video-cloud/ssai/all

Exécuter une trace pour une configuration d'annonce

Pour exécuter un suivi d'une configuration publicitaire, procédez comme suit :

Collecter des informations

Recueillez les informations suivantes pour le corps de votre demande d'API :

Paramètre Type Description
account_id Chaîne Identifiant du compte utilisateur
playback_config Objet Les champs sont définis dans le Détails du champ de configuration section du document de l'API SSAI.
title_metadata Objet Indique au système la durée de votre contenu afin de générer la bonne réponse publicitaire.

Par exemple, ce qui suit indique au système que le contenu dure 1 minute. Ceci est utilisé pour insérer des annonces en conséquence. 
  "title_metadata": {  "duration": "1m" }
videocloud_metadata Objet Facultatif.
Nécessaire uniquement si vous utilisez des variables de modèle qui font référence à des champs de métadonnées. Ceux-ci sont définis dans le Variables d'annonce section du document de l'API SSAI.

Requête

Créez une trace pour une configuration d'annonce SSAI.

Méthode POSTE
URL https://ssai.api.brightcove.com/v1/accounts/{account_id}/ssai_debug_vmap/debug.xml
En-têtes Autorisation : Porteur access_token (voir Obtenir des jetons d'accès)
Type de contenu : application/json
Corps de l'échantillon 
  {
    "playback_config":{
      "name": "config_name",
      "vmap_response_namespace": "config_namespace",
      "account_id": "account_id",
      "ad_config": {
        "enable_ads": true,
        "expected_ad_response": "dfp_ad_rules",
        "disable_server_beacons": false,
        "round_up_cue_points": false,
        "template_url": {
        "template": "template_url"
        }
      },
      "extend_beacon_guard_ttl": false
    },
    "title_metadata":{
      "duration": "39s"
    }
  }

Récupérer les détails de la trace

Une fois que vous avez exécuté une trace publicitaire comme indiqué dans la section précédente, vous pouvez récupérer des informations détaillées sur la trace à l'aide de l'ID de session.

Le session_id spécifie la session de mise en cache. Chaque session a sa propre durée en fonction de la longueur du contenu vidéo. Vous pouvez obtenir l'ID à partir de la réponse à l'appel d'API à l'étape précédente.

L'identifiant de session se trouve dans les réponses VMAP de la section précédente. Voici un exemple :

  <bc:BrightcoveDebug sessionID="your session id"></bc:BrightcoveDebug>

Requête

Pour récupérer les informations de suivi des appels d'annonces en fournissant un identifiant de session, utilisez une requête GET similaire à la suivante :

Méthode OBTENIR
URL https://ssai.api.brightcove.com/v1/accounts/{account_id}/ssai_traces/{session_id}/appels_annonces
En-têtes Autorisation : Porteur access_token (voir Obtenir des jetons d'accès)
Type de contenu : application/json
