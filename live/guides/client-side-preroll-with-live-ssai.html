    Publicités pré-roll côté client avec SSAI en direct

    Dans cette rubrique, vous apprendrez comment utiliser Brightcove Player pour lire une annonce pré-roll côté client avec des flux en direct activés pour l'insertion d'annonces côté serveur (SSAI).

    Présentation

    Lorsque vous diffusez des diffusions en direct avec l'insertion d'annonces côté serveur (SSAI) à l'aide de Brightcove Player, vous pouvez insérer une annonce pré-roll côté client. Pour les annonces côté client, cette fonctionnalité prend en charge les annonces IMA.

    Cette fonctionnalité nécessite les éléments suivants :

    • Plugin IMA3 v3.5.0+
    • Plugin SSAI v1.11.0+
    • Brightcove Player version 6.44.0+

    Avertissements

    • Les clients qui utilisent cette fonctionnalité doivent utiliser la livraison dynamique.
    • Seuls les plugins IMA et SSAI sont pris en charge (pas de FreeWheel).
    • Actuellement, cette fonctionnalité ne prend en charge qu'un seul joueur sur la page.

    Premiers pas

    Pour lire une annonce pré-roll IMA côté client avec un flux SSAI en direct, procédez comme suit :

    1. Créer un flux en direct compatible SSAI
    2. Créer un lecteur Brightcove
    3. Implémenter le pré-roll côté client à l'aide de Studio

    Créer un flux en direct compatible SSAI

    Le module Live prend en charge l'insertion d'annonces côté serveur (SSAI), ce qui permet aux annonces côté serveur d'être demandées et affichées pendant un flux en direct. Pour créer votre flux en direct, consultez les points suivants :

    • Implémentation des annonces côté serveur dans le module Live

    Créer un lecteur Brightcove

    Créez un nouveau lecteur Brightcove à l'aide de Video Cloud Studio. Pour plus de détails, reportez-vous à la section Démarrage rapide : Création et style d'un document Player .

    Vous devez utiliser une version 6.44.0+ de Brightcove Player.

    Implémentation du pré-roll côté client à l'aide de Studio

    Le moyen le plus simple de configurer votre lecteur pour les annonces de basculement automatique est avec Video Cloud Studio. Une fois que vous avez créé une configuration publicitaire et un lecteur, vous êtes prêt à configurer le lecteur pour le basculement automatique comme suit :

    1. Ouvrez le module PLAYERS et localisez le lecteur auquel vous souhaitez ajouter des fonctionnalités publicitaires.
    2. Sélectionnez le lien pour que le joueur ouvre ses propriétés.
    3. Dans le menu de navigation de gauche, sélectionnez Publicité.
    4. Cochez la case Activer côté client (IMA) .

    5. Incluez l'URL de votre balise publicitaire IMA. Pour cet exemple, nous utiliserons l'exemple d'URL de balise d'annonce.

      Pour plus d'informations sur les propriétés publicitaires du lecteur, consultez le document Configuration de la publicité du lecteur à l'aide du module Players .

    6. Cochez la case Activer le côté serveur (SSAI) .

      L'option côté serveur n'apparaîtra pas à moins que votre compte n'ait été activé pour SSAI.
    7. Dans le menu déroulant Sélectionner une configuration , sélectionnez la configuration d'annonce que vous souhaitez associer à ce lecteur.

    8. Si vous souhaitez que les superpositions s'affichent sur vos annonces, cochez la case Activer les superpositions d'informations publicitaires . Cela inclut les superpositions « En savoir plus » et le décompte des annonces.

      Les superpositions ne s'afficheront pas tant que vous ne les avez pas activées ici et configurées dans votre balise d'annonce VAST.
    9. Cliquez sur le bouton Enregistrer .

    10. Dans le menu de navigation de gauche, sélectionnez JSON Editor.

    11. Dans l'éditeur JSON, faites défiler vers le bas jusqu'à ce que vous voyez la ad_failover: true propriété.

    12. Remplacez la ad_failover: true propriété par ce qui suit : 

      "ima_preroll_with_ssai": true

    15. Pour publier le lecteur, sélectionnez Publier et intégrer > Publier les modifications.

    16. Maintenant, vous êtes prêt à publier votre événement en direct. Pour plus d'informations, consultez le document Implémentation des annonces côté serveur dans le document Live Module .
